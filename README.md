# HomeHQ

HomeHQ is a mobile-first Progressive Web App for organizing a shared house. It
centralizes household expenses, tasks, shopping, events, members, invitations,
permissions and balances in a single application that can be installed directly
from the browser.

The project was designed initially for a private shared household, while keeping
an architecture that can evolve into a public product.

## Product Overview

HomeHQ allows house members to:

- create and join shared houses;
- view a dashboard with upcoming events, bills, tasks, purchases and activity;
- split recurring or one-time bills between members;
- create tasks with multiple assignees;
- manage a shared shopping list and register completed purchases;
- track individual debts generated by purchases;
- perform controlled general balance adjustments between selected members;
- create events with date ranges, optional times and colored tags;
- invite members through email-restricted links or limited-use QR codes;
- configure section-level permissions for each member;
- choose a shared visual theme for each house.

## Highlights

### Progressive Web App

- Installable from Safari and other supported browsers.
- Standalone application experience without an app store installation.
- Service worker caching for faster repeated startup.
- Cached profile and house data while fresh data loads in the background.
- Responsive interface designed for desktop and mobile.
- Mobile safe-area support and fixed navigation header.

### Authentication and Profiles

- Email and password authentication through Supabase Auth.
- Persistent sessions.
- Unique email accounts.
- Required profile completion with username and gender.
- Optional profile images stored in Supabase Storage.
- Public profile information separated from private account data.

### House Permissions

Permissions are configured independently for each house section:

- `edit`: view and modify data;
- `view`: read-only access;
- `none`: section hidden from the user.

House owners always retain management access. Permission reads and updates use
secure PostgreSQL functions rather than trusting client-side checks.

### Financial Management

- Bills can be divided manually between members.
- Purchase debts are generated from custom member shares.
- Opposing debts remain as separate movements until a general adjustment is
  explicitly requested.
- General adjustments only consolidate debts between selected members.
- Debts involving non-selected members are never modified.
- Adjustment history preserves the original movements for auditability.

### Security

- Row Level Security enabled across application tables.
- Sensitive mutations implemented with PostgreSQL RPC functions.
- Invite validation by authenticated email.
- Expired and used invites are excluded.
- One active email invite per user and house.
- Normal users are limited to one house unless explicitly privileged.
- Environment credentials are excluded from source control.

## Architecture

The frontend is organized by domain:

```text
app/                         Next.js routes, layout and global styles
components/
  app/                       Root application composition
  auth/                      Profile and authentication flows
  dashboard/                 Home and house selection
  house/                     House-specific features
    events/                  Event calendar, forms and tag management
  shared/                    Reusable interface components
lib/                         Supabase client and shared utilities
types/                       Shared TypeScript domain models
supabase/
  migrations/                Versioned PostgreSQL schema and functions
public/                      PWA icons and service worker
```

The client is responsible for presentation and interaction. Data validation,
authorization, invite rules and financial consolidation are enforced in the
database.

## Local Setup

### Requirements

- Node.js 20 or newer
- npm
- A Supabase project

### Install dependencies

```bash
npm install
```

### Configure environment variables

Create `.env.local` from `.env.example`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Only use the public/publishable Supabase key in the frontend. Never expose the
Supabase service role key.

### Configure Supabase

Open the Supabase SQL Editor and execute every file inside
`supabase/migrations` in filename order.

The migrations create:

- profiles, houses and memberships;
- bills, tasks, events and shopping;
- debts and activity logs;
- invitations and permission rules;
- Row Level Security policies;
- secure database functions.

### Run the application

```bash
npm run dev
```

Open `http://localhost:3000`.

### Production build

```bash
npm run build
npm run start
```

## Installing as an App on iPhone

1. Open the deployed HomeHQ URL in Safari.
2. Select **Share**.
3. Select **Add to Home Screen**.
4. Open HomeHQ from the newly created icon.

## Database Changes

Database changes must be added as new timestamped SQL migration files. Existing
migrations should remain immutable after being applied to shared environments.
